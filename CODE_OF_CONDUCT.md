# 行動規範

agent-guild-orchestraは、背景、経験、立場にかかわらず、安心して技術的な協力ができる場を目指します。この文書は、本プロジェクトのために作成した行動規範です。

## 期待する行動

- 人ではなく、提案、コード、根拠、影響について議論する
- 相手の知識や意図を決めつけず、必要な前提を具体的に説明する
- 異論を歓迎し、再現手順、検証結果、参照情報を共有する
- 誤りを認め、訂正し、他者が安全に学べる余地を残す
- secret、個人情報、非公開情報を投稿しない
- セキュリティ問題を適切な非公開経路へ報告する

## 許容しない行動

- 侮辱、威圧、差別、嫌がらせ、個人攻撃
- 性的な言動や、望まれない接触
- 個人情報や非公開情報の公開、または公開をほのめかす行為
- 議論を意図的に妨害するspam、なりすまし、反復的な荒らし行為
- contributorへ権限外の操作、危険な情報開示、ライセンス違反を迫る行為

## 適用範囲

この行動規範は、リポジトリ、Issue、Pull Request、Discussion、その他本プロジェクトを代表する場に適用されます。プロジェクト外の行為でも、参加者の安全やプロジェクトの信頼に直接影響する場合は対応対象となることがあります。

## 報告と対応

行動規範違反を報告する場合は、該当するIssue、Pull Request、commentなどに表示されるGitHubの「Report content」機能を利用してください。報告対象の公開threadへ、個人情報、センシティブな詳細、非公開情報を書き込まないでください。本プロジェクト独自の非公開連絡先は、現在提供していません。セキュリティ脆弱性は[SECURITY.md](SECURITY.md)に従ってください。

「Report content」へ送られた報告は、GitHubのポリシーと機能に従って処理されます。maintainerは、リポジトリ上で確認できるcontentや行為に対し、注意、contentの編集・削除依頼、一時的な参加制限、恒久的な参加停止など、状況に応じた措置を取ることがあります。対応では必要以上に個人情報を共有しません。

この行動規範に関する判断は、project maintainerが本プロジェクトの健全性と参加者の安全を守るために行います。
